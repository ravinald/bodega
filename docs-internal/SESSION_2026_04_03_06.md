#### Features implemented:
- **4 source types**: apt, git, pypi, binary
- **Pipeline**: fetch → build → package → upload
- **Manifest-driven**: JSON manifests with `config_version: 1` envelope
- **S3 backend**: manifests stored in S3, loaded by default
- **TUI** (`bodega shell`): three-pane (Sources, Details, Log) with hotkeys
- **HTTP server** (`bodega serve`): serves APT repo, PEP 503 PyPI index, git bundles, binaries
- **API** (`/api/v1/`): packages, status, config endpoints
- **Build logging**: session log, per-package logs, audit.log, error.log in `/var/log/bodega/`
- **Git release mode** (default): downloads tarball instead of cloning
- **Per-type build roots**: configurable via `custom_paths` toggle
- **Checksum handling**: auto-detection from length (md5/sha1/sha256/sha512)
- **Create form**: type selector popup, URL auto-fill, Latest toggle, Validate source, raw JSON editor (Ctrl+T template, Ctrl+S apply)
- **Config editor** (`C` key): all settings editable, Ctrl+T defaults, Ctrl+R reset
